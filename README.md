# Hotel Recommendation System for Rhodes (Master Thesis)

A specialized recommendation engine developed for my Master's Thesis, designed to match travelers with the ideal hotel in Rhodes based on complex user preferences and location-based features.

## 🚀 Project Overview
The system uses Information Extraction (IE) and Semantic Web technologies to analyze hotel data and regional characteristics. By processing unstructured data from various travel websites and using a dedicated ontology for Rhodes' Points of Interest (POIs), the system calculates a satisfaction degree to provide personalized recommendations.

## 🛠 Tech Stack
* **Core Logic:** Content-Based Filtering Algorithms
* **Semantic Web:** Ontology-based representation & Reasoning techniques
* **Data Processing:** Information Extraction (IE) from unstructured text files
* **Frontend:** Web-based interface for User Preference input
* **Data Storage:** Structured processing and storage of extracted hotel/area features

## ✨ Key Contributions
* **Information Extraction:** Implementation of techniques to annotate and extract key features from hotel and area descriptions.
* **Ontology Engineering:** Use of an ontology to represent POIs and geographical areas of Rhodes, using reasoning to derive spatial context.
* **Hybrid Search Modes:** * *Quick Search:* Focused on basic user preferences for fast results.
    * *Detailed Search:* Advanced filtering for granular control over hotel and area features.
* **Scoring Algorithm:** Developed a custom algorithm based on the "satisfaction degree" of user preferences to rank the most suitable hotels.

## 📐 Methodology
1. **Data Collection:** Crawling and storing information from multiple travel websites.
2. **Feature Annotation:** Extracting structured data from text using IE.
3. **Semantic Reasoning:** Processing the Rhodes POI ontology to enrich the dataset.
4. **Matching Engine:** Applying content-based filtering to match extracted features with user-defined weights.
